## Category 1 — Code bugs / broken state

### BUG-1: `hooks.json` SessionStart timeout = 2s; installer writes 4s

**File:** `hooks/hooks.json` L10  
**Current value:** `"timeout": 2`  
**Installer value** (`scripts/installer.mjs` L214): `timeout: 4`  
**hooks.md** says: "hook-level budget is **2s** total" (wrong — aligns with hooks.json, not installer)

The installer (used by all real installs) patches `settings.json` with `timeout: 4`. The plugin-manifest `hooks.json` says `timeout: 2`. These diverge. The 4s is the intentional target: `session-start.mjs` spends up to 1.5s on git fetch + 0.5s on checkout + Node startup overhead. 2s is too tight; "SessionStart timeout fix" in v0.4.3 raised the installer value to 4 but never updated `hooks.json`.

**Fix:** Change `hooks.json` `"timeout": 2` → `"timeout": 4`

### docs-site/faq.md

**FAQ-1:** "What if git push fails?" answer (L27): "Wyren retries with `pull --rebase` up to 3 times."  
Wrong. Current implementation never calls `git pull --rebase`. It uses `_recoverNonFastForward()`: fetch, ancestry check, optional HEAD fast-forward, fresh wyren commit, retry. Three push attempts max, but no rebase.  
**Fix:** Rewrite the answer to describe actual behavior without the rebase reference.

**FAQ-2:** "How does Wyren handle merge conflicts?" answer (L79–80): "Wyren takes `--theirs` (the incoming version) and re-distills locally."  
This is stale from a pre-v0.4 implementation. `sync.mjs` no longer uses `--theirs`. On true divergence, it restores remote `.wyren/` files and returns `remote_diverged`; the user reconciles manually via `git fetch && git rebase`.  
**Fix:** Remove the `--theirs` sentence. Say: "If branches have diverged (user has local commits not on remote), Wyren leaves HEAD untouched, restores remote `.wyren/` files, and returns `remote_diverged`. Run `git fetch && git rebase origin/<branch>` then re-run `wyren distill --push`."
